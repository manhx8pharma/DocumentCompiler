Rất tốt, tôi sẽ phác thảo thêm sơ đồ kiến trúc tổng thể (System Architecture Diagram) cho tính năng Cross-template Field Mapping để bạn thấy rõ cách frontend, backend services và database phối hợp:

🔄 Luồng hoạt động
- Frontend (UI): người dùng thao tác trên trang Workflow Field Mapping (CRUD workflows, ánh xạ fields, xem preview).
- API Gateway: xử lý xác thực, phân quyền, điều phối request đến các service.
- Workflow Manager Service: quản lý cấu hình workflow, steps.
- Template Repository Service: cung cấp metadata templates và fields.
- Mapping Engine Service: thực hiện auto/manual mapping, transform, conflict resolution.
- Prefill Engine Service: lấy dữ liệu từ documents trước, điền vào bước sau.
- Document Service: lưu DocumentInstance và FieldValues.
- Provenance & Audit Service: ghi nguồn gốc dữ liệu và audit log.
- Database Layer: lưu trữ toàn bộ dữ liệu và quan hệ.

🎯 Ý nghĩa
- Frontend: tập trung vào trải nghiệm người dùng (quản lý workflows, ánh xạ, preview).
- Backend services: chia nhỏ theo chức năng (workflow, mapping, prefill, document, audit).
- Database: nền tảng lưu trữ, đảm bảo tính toàn vẹn và truy xuất nhanh.
- API Gateway: lớp trung gian bảo mật và điều phối.
