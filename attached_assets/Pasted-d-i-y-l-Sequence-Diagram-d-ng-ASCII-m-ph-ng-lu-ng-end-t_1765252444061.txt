🔎 Giải thích các bước
- Người dùng nhập dữ liệu ở Step 1 (Hợp đồng lao động).
- UI gửi request tạo DocumentInstance → API Gateway → Workflow Service.
- Workflow Service gọi Mapping Service để kiểm tra ánh xạ.
- Prefill Service chuẩn bị dữ liệu cho Step tiếp theo.
- Document Service lưu DocumentInstance và FieldValues.
- Provenance Service ghi nguồn gốc dữ liệu (manual-entry hoặc mapped).
- Audit Service ghi log hành động.
- DB lưu toàn bộ dữ liệu.
- UI hiển thị dữ liệu prefill cho người dùng review.
- Người dùng commit → Document Service cập nhật → Provenance + Audit ghi lại.

🎯 Ý nghĩa
- Sequence diagram cho thấy rõ tương tác tuần tự giữa các thành phần.
- Người dùng thao tác → UI → API Gateway → Services → DB → phản hồi về UI.
- Provenance và Audit luôn được ghi lại song song để đảm bảo minh bạch.
- Prefill Engine đóng vai trò trung gian, tự động điền dữ liệu từ bước trước sang bước sau.

👉 Với sơ đồ này, đội dev có thể dễ dàng hình dung luồng tuần tự end-to-end khi chạy một workflow.
